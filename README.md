# 🌐 Local Lynk

**Local Lynk** is an advanced full-stack e-commerce platform focused on connecting users with nearby products through location-aware discovery, real-time communication, and an immersive shopping experience. Built with a modern tech stack, the platform combines fast performance, responsive UI, advanced chat functionality, and a unique Demon Slayer inspired theme to create a highly interactive marketplace experience.

🔗 Live Demo: https://local-lynk.vercel.app/  
🔗 GitHub Repository: https://github.com/vikash1630/Local-Lynk

---

# 🚀 Features

## 📍 Nearby Product Discovery
- Detects and displays products based on user location.

- Reduces unnecessary browsing by prioritizing nearby listings.

- Real-time messaging between buyers and sellers.

# 🧑‍💻 How To Use

## 1️⃣ Clone The Repository

```bash
git clone https://github.com/vikash1630/Local-Lynk.git
```

---

## 2️⃣ Navigate Into The Project

```bash
cd Local-Lynk
```

---

## 3️⃣ Install Dependencies

```bash
npm install
```

---

## 4️⃣ Setup Environment Variables

Create a `.env` file in the root directory and configure the required environment variables.

Example:

```env
MONGODB_URI=your_mongodb_uri
JWT_SECRET=your_secret_key
CLOUDINARY_API_KEY=your_key
CLOUDINARY_SECRET=your_secret
```

---

## 5️⃣ Run The Development Server

```bash
npm run dev
```

---

## 6️⃣ Open In Browser

```bash
http://localhost:5173
```

# 🏗️ How It Was Built

## Frontend
- React.js for component-based architecture.
- Tailwind CSS for modern responsive styling.
- Advanced UI animations and interactive effects.
- Optimized state management for smoother rendering.

## Backend
- Node.js and Express.js for scalable API handling.
- REST APIs for authentication, products, and chat functionality.
- MongoDB used for storing users, products, and messages.

## Real-Time Communication
- Socket-based communication system for live chat updates.
- Reduced refresh dependency by enabling instant message syncing.

## Performance Optimizations
- Component optimization improved UI smoothness by approximately **30%** during navigation and interactions.
- Optimized rendering flow reduced unnecessary re-renders across product and chat modules.
- Lazy loading techniques improved initial page responsiveness by nearly **25%**.
- Efficient API handling reduced redundant requests and improved data fetching speed.
